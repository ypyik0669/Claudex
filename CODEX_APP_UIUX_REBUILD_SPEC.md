# Claudex UI/UX 重建规格

## 目标

Claudex 要做成中文优先的 Claude Code 桌面工作区，而不是静态 UI 或普通网页。界面行为参考 Codex App：低噪音、紧凑、以项目和会话为中心，所有功能都必须有真实本地状态或真实 CLI 命令支撑。

## 核心界面

- 左侧：项目、聊天、自动化、插件、设置。
- 中间：聊天、模型状态、输入框、快捷操作。
- 右侧：工作区、Claude Code、浏览器、终端、环境状态。
- 底部：输出、环境、变更、来源、子代理。

## 功能要求

- 默认通过本机 Claude Code CLI 运行。
- 直接 API 模式只能作为备用。
- 设置页能配置模型、基础 URL、权限、语言、字号和本地偏好。
- 插件、技能、MCP 和市场入口必须可见，并尽量通过 Claude Code CLI 获得真实输出。
- 文件编辑必须先审查差异再保存。
- 命令运行必须显示实时输出、退出码、工作目录和耗时。
- 遇到 Claude Code 原生权限确认或 slash command 流程时，必须提供交互式 Claude 终端入口。

## 中文化要求

- 默认语言为中文。
- “跟随系统”在当前公开包中也回落到中文，避免英文系统用户首次打开看到英文。
- 技术品牌名保留原文，例如 Claude Code、MCP、Git、IDE、API、URL。
- 普通产品文案、错误提示、快捷键说明、文档和 Release 说明必须使用中文。

## 验收

- `npm run build` 通过。
- 主界面、设置、插件、市场和快捷键弹窗显示中文。
- Release 页面和 README 显示中文。
- Windows 和 macOS Release 资产能自动生成。
