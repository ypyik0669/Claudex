## 这版不是 demo

这版默认不是自己模拟 Claude Code，而是调用你电脑里真正安装的 Claude Code CLI：

```text
claude 2.1.199 (Claude Code)
```

```text

```

同时保存到了 Claudex 本地聊天记录里。

## 第一次使用

1. 打开 `Claudex.exe`。
2. 左侧 `项目` 点 `+` 选择项目文件夹。
3. 中间输入框直接输入任务。
4. 默认执行方式是 `Claude Code`，会在当前项目目录里运行本机 `claude -p`。
5. 右侧 `Claude Code` 面板可以查看 Claude Code 状态、插件、MCP 和诊断。

## Claude Code 模式

在 `设置` 里可以看到这些设置：

- `执行方式`：默认 `Claude Code`；如果你想绕过 Claude Code 直接打 API，可以切到 `直接 API`。
- `Claude 命令`：默认 `claude`，程序会自动解析 Windows 下的真实 `claude.exe`。
- `权限模式`：对应 Claude Code 的 `--permission-mode`，支持 `default`、`acceptEdits`、`auto`、`plan`、`dontAsk`、`bypassPermissions`。
- `模型`：传给 Claude Code 的 `--model`，例如 `sonnet`、`opus` 等。
- `推理强度`：传给 Claude Code 的 `--effort`，支持 `low`、`medium`、`high`、`xhigh`、`max`。
- `高级 Claude Code`：可配置 agent、allowed/disallowed tools、tools、额外目录、MCP config、plugin dir/url、settings、setting sources、fallback model、预算、safe/bare、IDE、Chrome 和额外 CLI 参数。

`/model`、`/effort`、`/resume` 等 slash command 需要真实 TTY，所以请点击 `交互式 Claude` 打开 Claude Code TUI；普通 Claudex 聊天会使用对应的 CLI flags。

## 右侧 Claude Code 面板

右侧点击 `Claude Code` 后可以直接执行：

- `Auth`：运行 `claude auth status`。
- `插件`：运行 `claude plugin list`。
- `MCP`：运行 `claude mcp list`。
- `Doctor`：运行 `claude doctor`。
- 命令参考：可快速查看 `--help`、agents、project、plugin、marketplace、MCP。
- 自定义参数：例如输入 `plugin list`、`mcp list`、`auth status`、`agents --help`，点 `运行 Claude`。

这不是静态展示，会在当前项目路径真实执行 Claude Code 子命令。

现在右侧命令输出会实时流式显示 stdout/stderr，不用等命令结束才看到结果。

如果任务需要 Claude Code 原生交互，比如 permission prompt、slash command、插件登录流程，点击：

```text
交互式 Claude
```

它会在当前项目目录打开真正的 Claude Code TUI。

插件快捷操作：

- `市场`：查看插件市场。
- `插件名` 输入 `plugin@marketplace` 或插件名。
- `安装插件` / `更新` / `禁用`：直接调用真实 `claude plugin ...` 子命令。

## 工作区功能

右侧点击 `工作区`：

- 浏览当前项目文件树。
- 打开文本文件。
- 编辑文件时会显示 `未保存`。
- 保存前可展开 `差异预览` 看新增/删除行。
- `放弃更改` 可放弃当前编辑。
- 在当前项目里运行命令，例如 `npm test`、`node --version`，输出会实时流式显示。

为了安全：文件读写限制在当前项目目录内；超过 2MB 或二进制文件不会在编辑器里打开。

## 其他 UX

- `Ctrl+K`：命令面板。
- `Ctrl+N`：新聊天。
- `Ctrl+T`：切换 Browser 工具。
- `自动化`：保存稍后要运行的提示词。
- `插件`：Claudex 自己的插件/技能/工具偏好开关，会注入后续上下文；Claude Code 自身插件请用右侧 Claude Code 面板查看或执行。
- `浏览器`：用系统浏览器打开 URL。
- `终端`：在当前项目文件夹打开系统终端。

## 语言

默认中文。也可以在 `设置 -> 语言` 手动切换：

- `中文`
- `英文`

## Direct API 备用模式

如果你在 `设置 -> 执行方式` 切到 `直接 API`，可以直接配置：

- `OpenAI-compatible`
- `OpenRouter`
- `DeepSeek`
- `MiniMax`
- `Xiaomi MiMo`
- `LM Studio`
- `Anthropic`
- `Ollama / local`

OpenAI-compatible、OpenRouter、DeepSeek、MiniMax 默认使用 Bearer API key；Xiaomi MiMo 使用 `api-key` 请求头；LM Studio 和 Ollama 默认不需要 API key。API key 会由 Electron 主进程保存。Windows 支持 `safeStorage` 时会使用系统加密。

## Claude Code TUI 边界

当前已经接上 Claude Code 本体、子命令和常用 CLI flags，但不会伪造需要 TTY 的 Claude Code 原生交互：

- Claude Code 的交互式 permission prompt 还没有做成原生弹窗。
- 但已提供 `Interactive Claude` 入口，直接打开真实 Claude Code TUI 承接这类原生交互。
- `/model`、`/effort`、`/resume` 等 slash command 在真实 TUI 里使用；非交互聊天用设置里的 `--model`、`--effort` 等参数。
- 聊天回复和右侧命令面板都支持实时文本流。
- Workspace 已有文件级 diff preview；还没有 Claude Code 生成 patch 的逐块 accept/reject UI。
- 插件安装/更新/禁用已做成按钮流程；还没有完整 marketplace catalog UI。
