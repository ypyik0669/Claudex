## 这版不是 demo

这版默认不是自己模拟 Claude Code，而是调用你电脑里真正安装的 Claude Code CLI：

```text
claude 2.1.199 (Claude Code)
```

已验证：从 `Claudex.exe` 内部发送消息，实际走 Claude Code，并返回：

```text
CLAUDEX_DESKTOP_BRIDGE_OK
```

同时保存到了 Claudex 本地聊天记录里。

## 第一次使用

1. 打开 `Claudex.exe`。
2. 左侧 `Projects` 点 `+` 选择项目文件夹。
3. 中间输入框直接输入任务。
4. 默认执行方式是 `Claude Code`，会在当前项目目录里运行本机 `claude -p`。
5. 右侧 `Claude Code` 面板可以查看 Claude Code 状态、插件、MCP 和诊断。

## Claude Code 模式

在 `Settings` 里可以看到这些设置：

- `Execution`：默认 `Claude Code`；如果你想绕过 Claude Code 直接打 API，可以切到 `Direct API`。
- `Claude command`：默认 `claude`，程序会自动解析 Windows 下的真实 `claude.exe`。
- `Permission mode`：对应 Claude Code 的 `--permission-mode`，支持 `default`、`acceptEdits`、`auto`、`plan`、`dontAsk`、`bypassPermissions`。
- `Model`：传给 Claude Code 的 `--model`，例如 `sonnet`、`opus` 等。

## 右侧 Claude Code 面板

右侧点击 `Claude Code` 后可以直接执行：

- `Auth`：运行 `claude auth status`。
- `Plugins`：运行 `claude plugin list`。
- `MCP`：运行 `claude mcp list`。
- `Doctor`：运行 `claude doctor`。
- 自定义参数：例如输入 `plugin list`、`mcp list`、`auth status`，点 `Run Claude`。

这不是静态展示，会在当前项目路径真实执行 Claude Code 子命令。

现在右侧命令输出会实时流式显示 stdout/stderr，不用等命令结束才看到结果。

如果任务需要 Claude Code 原生交互，比如 permission prompt、slash command、插件登录流程，点击：

```text
Interactive Claude
```

它会在当前项目目录打开真正的 Claude Code TUI。

插件快捷操作：

- `Marketplace`：查看 marketplace。
- `Plugin name` 输入 `plugin@marketplace` 或插件名。
- `Install plugin` / `Update` / `Disable`：直接调用真实 `claude plugin ...` 子命令。

## 工作区功能

右侧点击 `Workspace`：

- 浏览当前项目文件树。
- 打开文本文件。
- 编辑文件时会显示 `Unsaved changes`。
- 保存前可展开 `Diff preview` 看新增/删除行。
- `Discard changes` 可放弃当前编辑。
- 在当前项目里运行命令，例如 `npm test`、`node --version`，输出会实时流式显示。

为了安全：文件读写限制在当前项目目录内；超过 2MB 或二进制文件不会在编辑器里打开。

## 其他 UX

- `Ctrl+K`：命令面板。
- `Ctrl+N`：新聊天。
- `Ctrl+T`：切换 Browser 工具。
- `Automations`（原 `Scheduled`，与官方 Codex App 命名对齐）：保存稍后要运行的提示词。
- `Plugins`：Claudex 自己的插件/技能/工具偏好开关，会注入后续上下文；Claude Code 自身插件请用右侧 Claude Code 面板查看或执行。
- `Browser`：用系统浏览器打开 URL。
- `Terminal`：在当前项目文件夹打开系统终端。

## 语言

默认跟随系统语言。也可以在 `Settings -> Language` 手动切换：

- `Follow system`
- `中文`
- `English`

## Direct API 备用模式

如果你在 `Settings -> Execution` 切到 `Direct API`，可以直接配置：

- `OpenAI-compatible`
- `Anthropic`
- `Ollama / local`

API key 会由 Electron 主进程保存。Windows 支持 `safeStorage` 时会使用系统加密。

## 本地数据

聊天记录、项目列表、设置保存在：

```text
%APPDATA%\Claudex\desktop-data.json
```

## 还没有完全替代 Claude Code TUI 的部分

当前已经接上 Claude Code 本体和子命令，但还不是完整复刻所有 TUI 交互：

- Claude Code 的交互式 permission prompt 还没有做成原生弹窗。
- 但已提供 `Interactive Claude` 入口，直接打开真实 Claude Code TUI 承接这类原生交互。
- 聊天回复和右侧命令面板都支持实时文本流。
- Workspace 已有文件级 diff preview；还没有 Claude Code 生成 patch 的逐块 accept/reject UI。
- 插件安装/更新/禁用已做成按钮流程；还没有完整 marketplace catalog UI。
